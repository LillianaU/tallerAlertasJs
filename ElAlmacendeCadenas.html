Código JavaScript

class Receta { constructor(nombre) { this.nombre = nombre; this.ingredientes = []; this.tiempo = 0; this.pasos = []; return this; // Enable chaining } agregarIngrediente(item, cantidad) { this.ingredientes.push({item, cantidad}); return this; // Key for chaining! } setTiempo(minutos) { this.tiempo = minutos; return this; } agregarPaso(instruccion) { this.pasos.push(instruccion); return this; } build() { return `Receta: ${this.nombre}...`; } }

Discusión: Orden de Métodos

Observa cómo cada método retorna this, permitiendo encadenar llamadas. El orden importa: primero inicializamos, luego configuramos, finalmente construimos.

Análisis del API Flexible

Ventajas del Chaining
  • Código más limpio y legible
  • Reducción de variables temporales
  • Flujo de trabajo intuitivo
Consideraciones
  • Debugging puede ser más difícil
  • Errores en cadena afectan todo
  • Acoplamiento de métodos
Casos de Uso
  • Constructores de consultas (Query Builders)
  • Configuración de objetos complejos
  • Manipulación del DOM (jQuery style)
¿Cómo funciona Method Chaining?

El Method Chaining es un patrón donde cada método retorna el objeto (this), permitiendo llamar métodos secuencialmente.

// Sin chaining: const receta = new Receta(); receta.agregarIngrediente("Harina"); receta.agregarIngrediente("Agua"); receta.setTiempo(30); // Con chaining: const receta = new Receta() .agregarIngrediente("Harina") .agregarIngrediente("Agua") .setTiempo(30);
