El Closure textChanged

El patrón textChanged utiliza un closure para encapsular el estado de cada campo de formulario:

createFieldValidator(fieldName)
Retorna función closure
Variables encapsuladas en el closure:
fieldName - Identifica el campo ('email' | 'password')
validators - Array de funciones de validación
state - { value, valid, touched, dirty }
listeners - Callbacks suscritos a cambios
Cada instancia mantiene su propio estado:
Closure Email state = { valid: true }
Closure Password state = { valid: false }
Ventaja: Cada campo tiene su propio estado independiente sin necesidad de clases o objetos globales. El closure "recuerda" el contexto en que fue creado.
